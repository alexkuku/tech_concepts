MongoDB 文档

BSON 格式、字段规则、点号表示法与数据类型

1. BSON 格式

BSON（Binary JSON）是 MongoDB 中文档的二进制表示格式。它基于 JSON 规范，但在 JSON 基础上扩展了更多的数据类型支持。

BSON 与 JSON 的关键区别

特性 JSON BSON
格式 文本格式 二进制格式
数据类型 6 种（string, number, boolean, null, array, object） 20+ 种（含 Date, ObjectId, Int32, Int64, Decimal128 等）
解析效率 需逐字符解析 带长度前缀，可快速跳过
字段顺序 规范未保证（但多数实现保留） 保证字段顺序

文档基本结构

MongoDB 文档由字段和值对组成，基本结构如下：

{
  field1: value1,
  field2: value2,
  field3: value3,
  ...
}
提示：字段的值可以是任何 BSON 类型，包括嵌入文档、数组以及文档数组。这种灵活的结构使 MongoDB 能够自然地表示层级数据。

2. 示例文档（含类型标注）

以下示例展示了一个包含多种 BSON 类型的典型 MongoDB 文档：

var mydoc = {
  _id: ObjectId("5099803df3f4948bd2f98391"),     // ObjectId 类型
  name: {                                        // 嵌入文档
    first: "Alan",
    last: "Turing"
  },
  birth: new Date('Jun 23, 1912'),               // Date 类型
  death: new Date('Jun 07, 1954'),               // Date 类型
  contribs: [                                    // 字符串数组
    "Turing machine",
    "Turing test",
    "Turingery"
  ],
  views: Long(1250000)                           // Int64 类型
}

类型标注说明

_id ObjectId 类型 -- 12 字节唯一标识符 name 嵌入文档 -- 包含 first 和 last 子字段 birth / death Date 类型 -- UTC 毫秒级时间戳 contribs 数组 -- 包含多个字符串元素 views Int64 (Long) 类型 -- 64 位整数

3. 字段名称规则

_id 字段

_id 是保留字段，用作文档的主键。它必须在整个集合中唯一，且不可变（一旦插入不可更改）。

字段名称限制

规则 说明
空字符限制 字段名称不能包含空字符（\0，即 null 字符）
美元符号 ($) 从 MongoDB 5.0 开始，字段名称可以包含 $ 和 .，但有一定限制。在早期版本中，顶级字段名不能以 $ 开头
点号 (.) 从 MongoDB 5.0 开始，允许在字段名中使用 .，但查询时需注意与点号表示法的冲突
唯一性 同一文档中字段名称必须唯一，不允许重复字段名

重复字段检测

MongoDB 在正常操作中不会主动检测重复字段名，但提供了以下方法进行验证：

  • validate 命令（MongoDB 6.1+）：使用 db.collection.validate() 可检测文档中的重复字段名
  • $objectToArray：使用聚合操作符 $objectToArray 将文档转为键值对数组，可间接检测是否存在重复键
警告：重复字段名可能导致不可预测的行为。虽然 MongoDB 不会阻止写入含有重复字段的文档（通过外部驱动或工具），但在读取时只会返回其中一个值，且具体返回哪个值是不确定的。

4. 点号表示法

点号表示法（Dot Notation）是 MongoDB 中访问嵌入文档字段和数组元素的核心语法。

访问数组元素

使用 "<array>.<index>" 语法访问数组中的特定元素，索引从 0 开始：

// 访问 contribs 数组的第一个元素
"contribs.0"    // 返回 "Turing machine"

// 访问 contribs 数组的第二个元素
"contribs.1"    // 返回 "Turing test"

访问嵌入文档字段

使用 "<embedded doc>.<field>" 语法访问嵌入文档中的字段：

// 访问 name 嵌入文档中的 first 字段
"name.first"   // 返回 "Alan"

// 访问 name 嵌入文档中的 last 字段
"name.last"    // 返回 "Turing"
提示：在查询和更新操作中，点号表示法必须用引号包裹。例如 {"name.first": "Alan"} 是正确的，而 {name.first: "Alan"} 会导致语法错误。

5. 文档限制

限制项 说明
最大文档大小 16 MB（包括所有嵌入文档和数组）。超过此限制需使用 GridFS
字段顺序 BSON 文档保证字段写入顺序（与 JavaScript 对象不同）。比较文档时字段顺序有意义
_id 位置 _id 字段始终是文档的第一个字段
投影中的字段顺序 查询引擎在投影操作中可能会重新排列字段顺序，不保证与原始文档一致
注意：字段顺序在比较操作中很重要。例如 {a: 1, b: 2} 与 {b: 2, a: 1} 在 BSON 比较中是不同的文档。这不同于 JavaScript 中对象属性顺序不保证的约定。

6. _id 字段规范

基本规则

  • 标准集合中 _id 字段是必需的
  • 如果在插入文档时未指定 _id，MongoDB 会自动生成一个 ObjectId
  • 集合创建时默认为 _id 建立唯一索引
  • _id 可以是除数组和正则表达式之外的任何 BSON 类型

常见 _id 生成策略

策略 类型 说明 适用场景
默认 ObjectId ObjectId 12 字节，包含时间戳、机器标识、进程 ID 和计数器 通用场景，无需手动管理
自然唯一键 字符串 / 数字 使用业务中天然唯一的字段（如用户名、邮箱） 有明确业务唯一标识的场景
自增序号 Int32 / Int64 使用计数器集合实现自增 ID 需要简洁递增 ID 的场景
UUID BSON BinData (子类型 4) 标准 128 位 UUID，存储为 BinData 分布式系统、跨系统唯一标识
驱动工具 UUID 取决于驱动 各语言驱动提供 UUID 工具类，可能使用不同 BinData 子类型 跨语言系统需注意子类型兼容性

ObjectId 结构

4 字节
时间戳 (Timestamp)
→
5 字节
随机值 (Random)
→
3 字节
递增计数器 (Counter)
注意：使用 UUID 作为 _id 时，不同语言驱动可能将 UUID 存储为不同的 BinData 子类型（子类型 3 或 4）。跨语言系统需确保统一子类型，否则查询可能不匹配。

7. 文档结构用途

在 MongoDB 中，"文档"不仅仅用于存储数据记录，还广泛应用于其他操作中：

{ }

查询过滤文档

用于 find()、update()、delete() 等操作的条件匹配

db.users.find({
  age: { $gte: 18 },
  status: "active"
})
$

更新规范文档

使用更新操作符（如 $set、$inc）指定修改内容

db.users.update(
  { _id: 1 },
  { $set: { status: "active" },
    $inc: { loginCount: 1 } }
)
1

索引规范文档

定义索引键和排序方向（1 为升序，-1 为降序）

db.users.createIndex({
  name: 1,
  age: -1
})
文档用途 结构特点 典型操作符
查询过滤 字段名 = 查询条件，支持条件操作符 $eq, $gt, $in, $and, $or
更新规范 字段名 = 更新操作符，操作符决定修改方式 $set, $inc, $push, $unset
索引规范 字段名 = 排序方向（1/-1），支持复合索引 1（升序）, -1（降序）, "2d", "2dsphere"

8. BSON 数据类型

BSON 支持丰富的数据类型，每种类型都有对应的编号（用于 $type 操作符）和别名：

类型 编号 别名 说明
Double 1 "double" 64 位双精度浮点数
String 2 "string" UTF-8 编码字符串
Object 3 "object" 嵌入文档
Array 4 "array" 值列表，可包含多种类型
BinData 5 "binData" 二进制数据，含子类型标识
ObjectId 7 "objectId" 12 字节唯一文档标识符
Boolean 8 "bool" true 或 false
Date 9 "date" UTC 毫秒级时间戳
Null 10 "null" 空值
Regex 11 "regex" 正则表达式，含选项标志
Int32 16 "int" 32 位有符号整数
Timestamp 17 "timestamp" 内部时间戳类型（用于复制）
Int64 18 "long" 64 位有符号整数
Decimal128 19 "decimal" 128 位十进制浮点数（精确数值计算）
提示：可使用 $type 操作符按类型查询文档，例如 { age: { $type: "int" } } 查询 age 字段为 Int32 类型的文档。编号 6（未定义）和 12 到 15 为已废弃或内部类型，不在上表中列出。

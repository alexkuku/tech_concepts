MongoDB 聚合操作

强大的数据处理与转换管道

聚合概述

聚合操作处理多个文档并返回计算后的结果。聚合操作将来自多个文档的值组合在一起， 对分组数据执行各种运算，并返回单个结果。MongoDB 提供了两种执行聚合的方式：

⚙

聚合管道 推荐

基于管道的数据处理范式，将文档通过多个阶段（Stage）依次处理， 每个阶段的输出作为下一个阶段的输入，功能强大且灵活。

★

单用途聚合方法

提供简单的聚合操作，如文档计数（count）、去重（distinct）等， 适用于简单的聚合需求，缺乏聚合管道的灵活性和功能。

推荐做法：聚合管道是 MongoDB 中执行聚合的首选方法， 它提供了比单用途聚合更丰富的功能和更好的性能控制能力。

聚合管道

聚合管道由多个阶段（Stage）组成，每个阶段对文档流进行一种操作转换。 文档依次通过管道中的每个阶段，前一个阶段的输出作为后一个阶段的输入， 最终产生聚合结果。

管道工作原理

  • 管道接收一个文档流作为输入
  • 每个阶段对文档执行一种操作，生成新的文档流
  • 阶段之间顺序执行，前阶段的输出是后阶段的输入
  • 同一阶段可以出现多次（如多个 $match 或 $group）
  • 管道可以通过 $facet 实现多分支并行处理
理解要点：管道中的每个阶段都会对文档流进行转换， 这意味着文档在经过每个阶段后可能会被过滤、重塑、分组或组合。 合理的阶段顺序对性能至关重要。

常用管道阶段

以下是聚合管道中最常用的阶段操作，每个阶段对文档执行特定的转换操作：

阶段 说明 示例
$match 过滤文档，只传递匹配条件的文档到下一阶段 {"$match": {status: "A"}}
$group 按指定字段分组并对每组执行聚合计算 {"$group": {_id: "$city", total: {"$sum": 1}}}
$sort 对文档按指定字段排序 {"$sort": {total: -1}}
$limit 限制传递到下一阶段的文档数量 {"$limit": 5}
$skip 跳过指定数量的文档 {"$skip": 10}
$project 投影/重塑文档结构，包含、排除或重命名字段 {"$project": {name: 1, _id: 0}}
$unwind 将数组字段展开为多个文档，每个数组元素生成一个文档 {"$unwind": "$tags"}
$lookup 左外连接，从另一个集合中关联文档 {"$lookup": {from: "orders", ...}}
$addFields / $set 向文档添加新字段或覆盖现有字段的值 {"$addFields": {total: {"$add": ["$a","$b"]}}}
$count 计算当前文档数量并返回计数字段 {"$count": "totalCount"}
$facet 在同一输入文档集上运行多个子管道，实现多维度聚合 Multiple sub-pipelines
$bucket 按指定边界将文档分桶分类 Auto bucket by boundaries
$out 将管道结果写入指定集合（替换已有内容） Output to new collection
$merge 将管道结果合并到已有集合（可更新或插入） Merge/update existing collection
注意：$out 和 $merge 必须是管道的最后一个阶段。 $out 会完全替换目标集合的内容，而 $merge 则可以增量更新目标集合。

完整管道示例

以下示例展示如何使用聚合管道查找电影数据库中作品数量最多的前 3 位导演：

需求分析

  1. 过滤掉没有导演信息的电影
  2. 将 directors 数组展开（每部电影可能有多个导演）
  3. 按导演分组并计算每人的电影数量
  4. 按电影数量降序排序
  5. 取前 3 名

聚合管道代码

db.movies.aggregate([
  // 阶段1: 过滤 - 只保留有导演字段的文档
  { $match: { directors: { $exists: true, $ne: null } } },

  // 阶段2: 展开 - 将 directors 数组拆分为多个文档
  { $unwind: "$directors" },

  // 阶段3: 分组 - 按导演分组，计算每组的电影数量
  { $group: {
      _id: "$directors",
      movieCount: { $sum: 1 }
  }},

  // 阶段4: 排序 - 按电影数量降序排列
  { $sort: { movieCount: -1 } },

  // 阶段5: 限制 - 只取前 3 名
  { $limit: 3 }
])

聚合表达式

聚合操作支持丰富的表达式，用于在管道阶段中对字段进行计算、比较和转换。 以下是按类别分组的主要表达式：

累加器表达式（Accumulators）

主要用于 $group 阶段，对分组内的值进行汇总计算：

$sum $avg $min $max $push $addToSet $first $last

比较表达式（Comparison）

比较两个值并返回布尔结果：

$eq $gt $lt $gte $lte $ne

算术表达式（Arithmetic）

执行数值计算操作：

$add $subtract $multiply $divide $mod

字符串表达式（String）

对字符串进行操作和转换：

$concat $substr $toLower $toUpper $split

日期表达式（Date）

从日期字段中提取时间组成部分：

$year $month $dayOfMonth $hour $minute $second

条件表达式（Conditional）

根据条件返回不同的值：

$cond $ifNull $switch

表达式使用示例

// $cond - 条件表达式
{ $project: {
    level: {
      $cond: { if: { $gte: ["$score", 80] }, then: "优秀", else: "继续努力" }
    }
}}

// $ifNull - 空值替换
{ $project: {
    displayName: { $ifNull: ["$nickname", "$realName"] }
}}

// $switch - 多条件分支
{ $project: {
    grade: {
      $switch: {
        branches: [
          { case: { $gte: ["$score", 90] }, then: "A" },
          { case: { $gte: ["$score", 80] }, then: "B" },
          { case: { $gte: ["$score", 70] }, then: "C" }
        ],
        default: "D"
      }
    }
}}

$lookup 连接操作

$lookup 是聚合管道中实现左外连接（Left Outer Join）的阶段， 用于从另一个集合中关联文档。MongoDB 提供了两种 $lookup 语法： 基本连接和管道连接。

基本连接（Equality Match）

最简单的连接方式，通过指定字段值进行等值匹配：

{
  $lookup: {
    from: "orders",           // 被连接的集合
    localField: "userId",     // 当前集合的连接字段
    foreignField: "_id",      // 目标集合的连接字段
    as: "userOrders"          // 输出数组字段名
  }
}

执行流程：对当前集合中每个文档，从 from 集合中查找 foreignField 等于 localField 的所有文档， 结果以数组形式放入 as 指定的字段。

管道连接（Pipeline Lookup）

支持更复杂的连接条件，可在 $lookup 内部使用子管道：

{
  $lookup: {
    from: "orders",
    let: { userId: "$_id", status: "$status" },  // 定义变量
    pipeline: [
      { $match:
        { $expr:
          { $and: [
            { $eq: ["$userId", "$$userId"] },     // 引用外部变量
            { $eq: ["$status", "$$status"] }
          ]}
        }
      },
      { $project: { orderId: 1, amount: 1, _id: 0 } }
    ],
    as: "matchingOrders"
  }
}

非关联子查询（Uncorrelated Subquery）

子管道不引用外部文档的变量，对每个输入文档执行相同的查询：

{
  $lookup: {
    from: "products",
    pipeline: [
      { $match: { category: "electronics" } },
      { $sort: { price: -1 } },
      { $limit: 5 }
    ],
    as: "topElectronics"
  }
}
提示：非关联子查询的结果对所有输入文档都相同， MongoDB 5.1+ 会自动优化为只执行一次子查询。

关联子查询（Correlated Subquery）

子管道通过 let 和 $expr 引用外部文档的字段， 对每个输入文档执行不同的查询：

{
  $lookup: {
    from: "comments",
    let: { postId: "$_id" },
    pipeline: [
      { $match:
        { $expr: { $eq: ["$postId", "$$postId"] } }
      },
      { $sort: { createdAt: -1 } },
      { $limit: 3 }
    ],
    as: "recentComments"
  }
}
性能注意：关联子查询会对每个输入文档单独执行子管道， 当输入文档数量较大时可能产生性能问题。建议在子管道中尽早使用 $match 并确保相关字段有索引。

单用途聚合方法

MongoDB 提供了一些简单的聚合方法，适用于不需要复杂管道的常见聚合场景。 这些方法语法简洁，但功能有限。

db.collection.estimatedDocumentCount()

返回集合的近似文档数量，基于集合的元数据快速返回结果。

// 获取 users 集合的近似文档数
db.users.estimatedDocumentCount()
适用场景：不需要精确计数时使用，速度极快。 不适用于分片集群或有孤立文档的集合。

db.collection.countDocuments(query)

返回匹配查询条件的文档精确数量，内部使用聚合管道实现。

// 统计状态为 active 的用户数
db.users.countDocuments({ status: "active" })

// 统计所有文档
db.users.countDocuments({})

db.collection.distinct(field, query)

返回指定字段的所有去重值，可配合查询条件使用。

// 获取所有不同的城市
db.users.distinct("city")

// 获取活跃用户的不同城市
db.users.distinct("city", { status: "active" })
方法 精确度 速度 支持查询条件
estimatedDocumentCount() 近似 极快 不支持
countDocuments() 精确 较慢 支持
distinct() 精确 中等 支持

性能优化建议

合理设计聚合管道对性能至关重要。以下是一些经过验证的最佳实践：

✓
尽早使用 $match 过滤数据
将 $match 放在管道的前面可以尽早减少文档数量， 降低后续阶段的处理开销。如果 $match 是管道的第一个阶段， MongoDB 可以利用索引来加速查询。
✓
在 $match 上使用索引
确保 $match 阶段使用的查询字段已建立索引。 管道开头的 $match 可以利用索引扫描来快速过滤文档， 避免全集合扫描。
✓
使用 $project 减少文档大小
在数据密集的管道中，使用 $project 或 $unset 尽早移除不需要的字段，减少传递到后续阶段的文档大小， 降低内存占用和网络传输开销。
✓
使用 $facet 实现多维度聚合
$facet 允许在同一输入文档集上并行运行多个子管道， 适合需要同时获取多个维度聚合结果的场景（如同时获取按分类、按时间、 按地区的统计结果），避免多次执行管道。

常见性能陷阱

避免以下做法：
  • 将 $sort 放在 $match 之前 -- 会对全部文档排序而非仅过滤后的文档
  • 在 $group 之前缺少 $match -- 会导致对大量无用数据进行分组计算
  • 不加限制地使用 $unwind -- 大数组展开会生成海量中间文档
  • 忽略 100MB 内存限制 -- 大数据量管道需使用 allowDiskUse: true

MongoDB 概述

面向现代应用的文档数据库

什么是 MongoDB

MongoDB 是一款面向现代应用的文档数据库，以灵活的 JSON 类文档（BSON）形式存储数据。 与传统关系数据库不同，MongoDB 无需 ORM 映射，文档模型天然契合应用代码中的对象结构， 使开发更加高效直观。

灵活模式，无需 ORM

MongoDB 的文档模型允许不同文档拥有不同的字段结构，无需预先定义固定的表模式。 这种灵活性使得应用迭代更加快速，尤其在需求频繁变更的场景下优势明显。 数据以字段-值对（field-value pairs）形式存储，嵌入式文档和数组可直接在一条记录中表达层级关系， 大幅减少联表查询（JOIN）的需求。

核心架构组件

1. 文档数据库（Document Database）

MongoDB 以文档作为基本数据单元。每条记录是一组字段-值对的集合，以 BSON （Binary JSON）格式存储。文档中可以包含嵌入式文档和数组，从而以自然的方式表达 层级数据关系，显著减少联表操作的需求。

提示：BSON 是 JSON 的二进制扩展格式，支持更多数据类型（如 Date、BinData、Decimal128 等）， 同时提供更高效的遍历和解析性能。

2. 事务（Transactions）

MongoDB 支持多文档 ACID 事务，确保跨多个文档和集合的操作具有原子性、一致性、隔离性和持久性。 在分片集群中，MongoDB 还支持分布式事务，可跨多个分片执行事务操作，保障全局数据一致性。

说明：单文档操作天然具有原子性。只有在需要跨多个文档或集合保持一致性时， 才需要显式使用多文档事务。

3. 高可用（High Availability）

MongoDB 通过副本集（Replica Set）实现高可用性。副本集由一组维护相同数据集的 mongod 实例组成，包含一个主节点（Primary）和多个从节点（Secondary）。当主节点发生故障时， 系统自动进行故障转移（Automatic Failover），选举新的主节点，确保服务连续不中断。 数据冗余通过复制机制实现，从节点持续同步主节点的数据。

4. 水平扩展（Horizontal Scaling）

MongoDB 通过分片（Sharding）实现水平扩展，将数据分布到多个服务器（分片）上。 分片集群自动进行数据分布，根据分片键将数据均衡地分散到各分片上。 此外，区域分片（Zone Sharding）允许将数据定向到特定分片，满足数据本地化或合规性需求。

注意：选择合适的分片键至关重要。分片键一旦确定便不可修改， 不当的分片键可能导致数据倾斜或性能瓶颈。

MongoDB 与关系数据库对比

下表列出了 MongoDB 核心概念与传统关系数据库的对应关系，帮助有 RDBMS 背景的开发者快速理解 MongoDB 的数据模型。

MongoDB 关系数据库
文档（Document） 行（Row）
集合（Collection） 表（Table）
字段（Field） 列（Column）
嵌入式文档（Embedded Document） 连接（JOIN）
_id 主键（Primary Key）
灵活模式 固定模式
核心差异：关系数据库要求预定义模式（Schema），所有行必须遵守相同的列结构； 而 MongoDB 的灵活模式允许同一集合中的文档拥有不同的字段，非常适合敏捷开发和快速迭代。

连接 MongoDB

连接字符串格式

MongoDB 使用连接字符串（Connection String）指定连接参数。标准格式如下：

/* 标准连接字符串 */
mongodb://user:password@host1:port1,host2:port2/database?options

/* SRV 连接字符串（推荐） */
mongodb+srv://user:password@cluster.example.com/database?options

各组成部分说明：

  • mongodb:// — 标准协议前缀，指定主机和端口
  • mongodb+srv:// — SRV 协议前缀，通过 DNS SRV 记录自动发现集群节点（推荐）
  • user:password — 认证凭据
  • host:port — 数据库实例地址，可指定多个实现高可用
  • database — 连接时默认使用的数据库，也是认证数据库（未指定 authSource 时）
  • options — 连接选项参数，如 authSource、replicaSet、readPreference 等

特殊字符的百分号编码

当用户名或密码中包含特殊字符时，必须使用百分号编码（Percent Encoding）进行转义， 否则连接字符串解析将出现错误。

特殊字符 编码值 示例
@ %40 密码 p@ss 写为 p%40ss
: %3A 密码 p:ss 写为 p%3Ass
/ %2F 密码 p/ss 写为 p%2Fss
% %25 密码 p%ss 写为 p%25ss
+ %2B 密码 p+ss 写为 p%2Bss
空格 %20 密码 p ss 写为 p%20ss
重要：如果密码中包含 @、:、% 等字符而未进行编码， MongoDB 会将其误认为连接字符串的分隔符，导致连接失败或认证错误。

💡 学习建议

  • 循序渐进：按照学习路径的顺序进行学习，每个概念都建立在前一个概念的基础上
  • 动手实践：在学习每个概念后，使用 mongosh 进行实际操作加深理解
  • 理解数据模型：MongoDB 的文档模型与传统关系型数据库差异较大，务必深入理解
  • 关注查询优化：索引和聚合是 MongoDB 性能的关键，花时间掌握这两个核心主题
  • 实践场景：思考每个概念在实际应用场景中的作用，如高可用、水平扩展等
  • 版本关注：注意 MongoDB 不同版本之间的差异和新特性，特别是事务和分片的演进

📖 核心术语速查

术语中文简要说明
BSON二进制 JSONMongoDB 的数据存储格式，扩展了 JSON 支持更多数据类型
Document文档MongoDB 中数据的基本单元，类似关系型数据库的行
Collection集合文档的容器，类似关系型数据库的表
Replica Set副本集一组维护相同数据集的 mongod 实例，提供高可用性
Shard分片存储数据子集的副本集，用于水平扩展
Shard Key分片键决定文档分布到哪个分片的字段或字段组合
Index索引提高查询效率的特殊数据结构
Aggregation Pipeline聚合管道数据处理管道框架，通过阶段变换文档
Oplog操作日志记录所有写操作的特殊固定大小集合，用于复制
Read Concern读关注控制读操作的一致性和隔离级别
Write Concern写关注控制写操作的确认级别和持久性保证
Read Preference读取偏好决定读操作路由到副本集的哪个成员
Capped Collection固定大小集合固定大小的集合，按插入顺序自动覆盖旧文档
Replication Lag复制延迟次节点数据落后主节点的时间差
Chunk数据段分片中数据划分和迁移的最小单元
Config Server配置服务器存储分片集群元数据的特殊副本集
mongos查询路由器分片集群的入口，负责将查询路由到正确的分片
CRUD增删改查Create、Read、Update、Delete 四种基本数据操作
Change Stream变更流基于 oplog 的实时数据变更通知机制
TTL IndexTTL 索引自动删除过期文档的特殊索引

🎯 mongosh 常用命令

功能命令说明
查看数据库show dbs列出所有数据库
切换数据库use <db>切换到指定数据库
查看集合show collections列出当前数据库的所有集合
插入文档db.coll.insertOne({...})向集合中插入一个文档
查询文档db.coll.find({...})按条件查询集合中的文档
更新文档db.coll.updateOne({...}, {$set: {...}})更新匹配条件的单个文档
删除文档db.coll.deleteOne({...})删除匹配条件的单个文档
创建索引db.coll.createIndex({field: 1})在指定字段上创建索引
查看索引db.coll.getIndexes()列出集合的所有索引
聚合查询db.coll.aggregate([...])执行聚合管道操作

💾 存储

掌握持久化存储和卷管理

存储概述

Kubernetes 的存储系统为 Pod 提供持久化存储支持，确保容器重启或迁移时数据不丢失。存储系统通过 Volume 抽象层支持多种存储后端。

为什么需要持久化存储？

  • 容器是临时的：容器重启时数据会丢失
  • 应用需要持久化：数据库、文件存储等需要持久化
  • 多容器共享：多个容器可能需要共享数据
  • 状态迁移：Pod 迁移时需要保留状态

↓ 绑定到 ↓
PV
PVC
↓ 使用 ↓

Volume 核心概念

Volume 是 Kubernetes 中存储数据的目录，可以被 Pod 中的容器访问。Volume 的生命周期与 Pod 相同。

Volume 的类型

类型 说明 持久化 使用场景
emptyDir 节点上临时目录 否 临时数据、缓存
hostPath 节点主机文件系统 依赖节点 系统监控、日志
PV/PVC 持久化卷 是 数据库、持久化应用
configMap 配置数据 是 注入配置文件
secret 敏感数据 是 注入证书、密钥
downwardAPI Pod 信息 否 注入 Pod 元数据
projected 多源卷 是 组合多个卷

Volume 使用示例

# 使用 emptyDir
apiVersion: v1
kind: Pod
metadata:
  name: test-pd
spec:
  containers:
  - name: test-container
    image: nginx
    volumeMounts:
    - mountPath: /cache
      name: cache-volume
  volumes:
  - name: cache-volume
    emptyDir:
      sizeLimit: 500Mi  # 可选：限制大小

# 使用 hostPath
volumes:
- name: host-path-volume
  hostPath:
    path: /var/log  # 节点主机路径
    type: Directory  # 类型：Directory、File、Socket 等

emptyDir 应用场景

场景 说明
共享数据 同一 Pod 内多个容器共享数据
临时文件 不需要持久化的临时数据
缓存 加速访问的缓存数据
排序中间文件 批处理任务的中间结果

PV 和 PVC 核心概念

PersistentVolume (PV) 是集群级别的存储资源，PersistentVolumeClaim (PVC) 是命名空间级别的存储请求。

Pod
↑ 使用 ↑
PVC (PersistentVolumeClaim)
↑ 绑定 ↑
PV (PersistentVolume)
↑ 提供存储 ↑
物理存储 (NFS/云存储等)

PV (PersistentVolume)

PV 是集群级别的存储资源，独立于 Pod 存在。PV 由集群管理员创建或通过 StorageClass 动态创建。

apiVersion: v1
kind: PersistentVolume
metadata:
  name: my-pv
spec:
  capacity:
    storage: 10Gi  # 存储容量
  accessModes:
    - ReadWriteOnce  # 访问模式
  persistentVolumeReclaimPolicy: Delete  # 回收策略
  storageClassName: manual  # 存储类
  nfs:  # 存储类型
    server: 192.168.1.100
    path: /data/k8s

PV 访问模式

模式 缩写 说明 支持存储
ReadWriteOnce RWO 单个节点读写 NFS、Ceph、AWS EBS、GCE PD 等
ReadOnlyMany ROX 多个节点只读 NFS、Ceph 等
ReadWriteMany RWX 多个节点读写 NFS、Ceph、GlusterFS 等
ReadWriteOncePod RWOP 单个 Pod 读写（v1.22+） 块存储（如 AWS EBS）

PV 回收策略

策略 PVC 删除时 数据保留
Retain PV 状态变为 Released 保留，需手动清理
Delete PV 和底层存储被删除 删除，不保留
Recycle 执行 rm -rf /volume/* 删除数据后可重用
注意：Recycle 策略已在 v1.15 废弃，建议使用动态 provisioning。

PVC (PersistentVolumeClaim)

PVC 是用户对存储的请求，类似于 Pod 对计算资源的请求。PVC 会自动绑定到满足条件的 PV。

apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: my-pvc
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 5Gi  # 请求的存储容量
  storageClassName: fast  # 可选：指定存储类

PVC 和 PV 绑定流程

1. 用户创建 PVC
↓
2. K8s Watch 到 PVC 请求
↓
3. 匹配满足条件的 PV
↓
4. PVC 和 PV 绑定（一对一）
↓
5. Pod 可以使用 PVC

Pod 使用 PVC

apiVersion: v1
kind: Pod
metadata:
  name: mypod
spec:
  containers:
  - name: myfrontend
    image: nginx
    volumeMounts:
    - mountPath: "/var/www/html"
      name: mypd
  volumes:
  - name: mypd
    persistentVolumeClaim:
      claimName: my-pvc  # 引用 PVC

静态和动态供给

Kubernetes 支持两种 PV 供给方式：静态供给和动态供给。

静态供给

管理员预先创建 PV，PVC 请求时绑定到匹配的 PV。

管理员创建 PV
↓
用户创建 PVC
↓
PVC 绑定到匹配的 PV

动态供给

通过 StorageClass 自动创建 PV，无需管理员预先创建。

管理员创建 StorageClass
↓
用户创建 PVC（指定 StorageClass）
↓
自动创建 PV
↓
PVC 绑定到新创建的 PV

静态 vs 动态供给

特性 静态供给 动态供给
创建方式 管理员手动创建 自动创建
灵活性 低 高
效率 低 高
适用场景 小规模、固定存储 大规模、灵活需求

StorageClass 重要

StorageClass 定义了存储的类型和参数，用于动态创建 PV。它类似云服务中的虚拟机规格（如 t2.micro、t2.large）。

StorageClass 的组成

字段 说明
provisioner 卷创建程序（如 kubernetes.io/aws-ebs）
parameters 存储后端特定的参数
reclaimPolicy 回收策略（默认为 Delete）
mountOptions 挂载选项
volumeBindingMode 卷绑定模式
allowedTopologies 允许的节点拓扑

StorageClass 示例

# AWS EBS StorageClass
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: aws-ebs-sc
provisioner: kubernetes.io/aws-ebs
parameters:
  type: gp2  # 卷类型
  iopsPerGB: "10"  # IOPS
  fsType: ext4  # 文件系统类型
reclaimPolicy: Delete
volumeBindingMode: WaitForFirstConsumer

# NFS StorageClass (需要 NFS provisioner)
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: nfs-sc
provisioner: example.com/nfs
parameters:
  archiveOnDelete: "false"
reclaimPolicy: Delete

默认 StorageClass

可以指定一个 StorageClass 为默认，PVC 不指定时自动使用。

# 设置默认 StorageClass
kubectl patch storageclass standard -p '{"metadata": {"annotations":{"storageclass.kubernetes.io/is-default-class":"true"}}}'

volumeBindingMode

模式 说明 优点
Immediate 立即绑定（PVC 创建时） 响应快
WaitForFirstConsumer 使用 Pod 时绑定（延迟绑定） 根据 Pod 调度选择合适节点

卷快照

卷快照（Volume Snapshot）是卷在特定时间点的副本，可用于备份、恢复或克隆卷。

快照类型

类型 说明
VolumeSnapshot 快照请求，类似 PVC
VolumeSnapshotContent 实际快照内容，类似 PV
VolumeSnapshotClass 快照类，定义快照参数

创建快照

# VolumeSnapshotClass
apiVersion: snapshot.storage.k8s.io/v1
kind: VolumeSnapshotClass
metadata:
  name: csi-hostpath-snapclass
driver: hostpath.csi.k8s.io
deletionPolicy: Delete

# 创建快照
apiVersion: snapshot.storage.k8s.io/v1
kind: VolumeSnapshot
metadata:
  name: my-snapshot
spec:
  volumeSnapshotClassName: csi-hostpath-snapclass
  source:
    persistentVolumeClaimName: my-pvc  # 基于哪个 PVC 创建快照

从快照恢复 PVC

apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: restored-pvc
spec:
  accessModes:
    - ReadWriteOnce
  storageClassName: csi-hostpath-sc
  resources:
    requests:
      storage: 10Gi
  dataSource:
    name: my-snapshot
    kind: VolumeSnapshot
    apiGroup: snapshot.storage.k8s.io

PV 克隆

PV 克隆允许你创建现有卷的副本。这个特性需要在 PVC 中指定dataSource。

apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: cloned-pvc
spec:
  accessModes:
    - ReadWriteOnce
  storageClassName: csi-hostpath-sc
  resources:
    requests:
      storage: 10Gi
  dataSource:
    name: original-pvc  # 要克隆的 PVC
    kind: PersistentVolumeClaim
注意：克隆操作需要 CSI 驱动支持。

本地临时存储

本地临时存储（Ephemeral Storage）用于 Pod 运行时的临时数据，包括容器镜像层、可写容器层、日志等。

本地临时存储来源

来源 说明
容器镜像层 容器镜像占用的空间
可写容器层 容器运行时写入的数据
日志 容器和 kubelet 日志
emptyDir 卷 emptyDir 卷的数据

管理本地临时存储

# API Server 配置
--eviction-hard=memory.available<100Mi,nodefs.available<10%,nodefs.inodesFree<5%,imagefs.available<15%,imagefs.inodesFree<10%

# Pod 限制
spec:
  containers:
  - name: mycontainer
    image: myimage
    resources:
      limits:
        ephemeral-storage: "2Gi"
      requests:
        ephemeral-storage: "1Gi"

临时存储驱逐策略

驱逐信号 说明
nodefs.available 节点文件系统可用空间
nodefs.inodesFree 节点文件系统可用 inode
imagefs.available 镜像文件系统可用空间
imagefs.inodesFree 镜像文件系统可用 inode

本章总结

关键要点：
  • Volume 是 Pod 中的存储抽象，生命周期与 Pod 相同
  • emptyDir 用于临时数据，hostPath 用于节点主机文件系统
  • PV 是集群级存储资源，PVC 是命名空间级存储请求
  • PV 访问模式包括 RWO、ROX、RWX、RWOP
  • PV 回收策略包括 Retain、Delete、Recycle
  • 动态供给通过 StorageClass 自动创建 PV
  • StorageClass 定义存储类型、参数和回收策略
  • 快照用于卷的备份和恢复
  • PV 克隆可以复制现有卷
  • 本地临时存储用于容器运行时的临时数据

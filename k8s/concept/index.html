💡 学习建议

  • 循序渐进：按照学习路径的顺序进行学习，每个概念都建立在前一个概念的基础上
  • 动手实践：在学习每个概念后，使用 kubectl 进行实际操作加深理解
  • 绘制图表：尝试自己绘制架构图和关系图，帮助理解概念之间的关系
  • 关注关键术语：Kubernetes 有很多专业术语，掌握这些术语对于深入学习非常重要
  • 实践场景：思考每个概念在实际应用场景中的作用和用法
  • 版本关注：注意 Kubernetes 不同版本之间的差异和新特性

📖 核心术语速查

术语 中文 简要说明
Pod 容器组 Kubernetes 最小的可部署单元
Node 节点 运行容器化应用程序的工作机器
Cluster 集群 由一组节点组成的 Kubernetes 系统
Control Plane 控制平面 管理集群状态的核心组件
Service 服务 为一组 Pod 提供稳定的网络访问方式
Deployment 部署 管理 Pod 副本和更新策略
PV/PVC 持久卷/持久卷声明 存储资源的抽象和声明式管理
kubectl kubectl Kubernetes 命令行工具

🎯 kubectl 常用命令

功能 命令 说明
查看节点 kubectl get nodes 列出集群中的所有节点
查看 Pod kubectl get pods 列出当前命名空间的所有 Pod
查看服务 kubectl get svc 列出所有服务
创建资源 kubectl apply -f file.yaml 从 YAML 文件创建或更新资源
删除资源 kubectl delete -f file.yaml 删除 YAML 文件中定义的资源
查看日志 kubectl logs 查看 Pod 的日志
进入容器 kubectl exec -it -- /bin/bash 进入 Pod 的容器中执行命令
描述资源 kubectl describe 显示资源的详细状态

理解 Kubernetes 的基本概念和架构

什么是 Kubernetes？

Kubernetes（常缩写为 K8s）是一个开源的容器编排平台，用于自动化部署、扩展和管理容器化应用程序。它最初由 Google 设计，现在由 Cloud Native Computing Foundation (CNCF) 维护。

Kubernetes 的核心目标：
  • 提供容器编排的高效管理
  • 实现应用程序的自动化部署和扩展
  • 提供自愈能力，自动处理故障恢复
  • 支持负载均衡和服务发现
  • 提供存储编排和网络管理
  • 支持自动滚动更新和回滚

为什么叫 K8s？

Kubernetes 这个名字中间有 8 个字母（ubernete），所以在技术社区中常被称为 K8s。

Kubernetes 组件

Kubernetes 集群由两种类型的节点组成：控制平面节点（Control Plane）和工作节点（Worker Node）。

控制平面组件

组件 功能描述
API Server 提供 RESTful API 接口，是所有组件的统一入口，处理所有内部和外部的 API 请求
etcd 高可用的键值存储，保存集群所有的配置数据和状态信息
Scheduler 负责调度新创建的 Pod 到合适的节点上运行，根据资源需求、策略等进行决策
Controller Manager 运行控制器进程，监控集群状态并确保实际状态与期望状态一致
Cloud Controller Manager 连接 Kubernetes 与云服务提供商的平台，管理节点、路由器等的生命周期

工作节点组件

组件 功能描述
Kubelet 节点代理，负责与控制平面通信，管理本节点上的 Pod 和容器
Kube-proxy 网络代理，维护节点上的网络规则，实现 Service 的负载均衡
容器运行时 负责运行容器的软件，如 Docker、containerd、CRI-O 等
用户提交请求到 API Server
↓
Controller Manager 监控状态变化
↓
Scheduler 分配 Pod 到节点
↓
Kubelet 在节点上创建和管理容器
↓
Kube-proxy 配置网络规则
↓
Pod 运行并更新状态到 etcd

Kubernetes 对象

Kubernetes 对象是 Kubernetes 系统中的持久化实体，用于表示集群的状态。Kubernetes 使用这些实体来表示集群的期望状态。

Kubernetes 对象
Pod
Node

Deployment
StatefulSet
DaemonSet
ConfigMap
Secret
PersistentVolume
Namespace

对象的基本特性

  • 期望状态：每当你创建一个对象时，都要告诉 Kubernetes 你希望它的状态是什么样的
  • 实际状态：Kubernetes 会持续工作的确保对象的实际状态与期望状态相同
  • 声明式 API：你只需要定义想要的状态，Kubernetes 会自动处理如何实现这些状态
类比理解：想象你是一个餐厅经理，你告诉服务员（Kubernetes）你希望有 10 个服务员在工作（期望状态），而不是亲自去招聘和安排。Kubernetes 会自动确保有 10 个服务员在工作，如果有服务员离职，它会自动招聘新的替补。

标签和选择器 核心概念

标签（Labels）是附加到 Kubernetes 对象（如 Pod）上的键值对，用于标识对象的属性。选择器（Selectors）用于选择具有特定标签的对象。

标签示例

metadata:
  labels:
    app: nginx
    environment: production
    tier: frontend

标签选择器

选择器类型 语法 说明 示例
等值选择器 key=value 选择键等于某个值的对象 environment=production
不等于 key!=value 选择键不等于某个值的对象 environment!=development
集合选择器 key in (values) 选择键在值集合中的对象 tier in (frontend, backend)
不存在 !key 选择没有某键的对象 !partition
标签的最佳实践：
  • 使用有意义的键和值，方便识别和管理
  • 避免使用对标签选择器没有价值的信息
  • 保持标签的一致性和可预测性
  • 使用命名约定，如 app、version、environment 等

命名空间 核心概念

命名空间（Namespace）是 Kubernetes 集群内部的一种抽象，用于将集群资源划分为逻辑组。它类似于一个虚拟集群，在同一个物理集群中可以创建多个虚拟集群。

命名空间的作用

  • 资源隔离：不同团队或项目可以使用不同的命名空间，避免资源冲突
  • 权限控制：可以针对不同命名空间设置不同的访问权限
  • 资源配额：可以为每个命名空间设置资源限制
  • 环境区分：可以用于区分开发、测试、生产环境

常见命名空间示例

命名空间 用途
default 默认命名空间，未指定命名空间的对象会被创建在这里
kube-system 由 Kubernetes 系统创建的对象
kube-public 所有用户（包括未认证用户）都可读的对象
kube-node-lease 用于节点心跳机制，提高集群性能
注意事项：并非所有资源都属于命名空间！某些资源（如 Node、PersistentVolume）是集群级别的，不属于任何命名空间。

注解

注解（Annotations）是附加到对象上的任意非识别数据。与标签类似，它们也是键值对，但具有不同的用途。

注解 vs 标签

特性 标签 (Labels) 注解 (Annotations)
主要用途 标识和选择对象 存储元数据
选择器支持 ✓ 支持 ✗ 不支持
数据大小 有限制（通常较小） 更大（最多 256KB）
查询效率 高效（用于筛选） 一般（用于存储）
场景 选择、分组、过滤 存储构建信息、调试信息、版本信息等

常见注解用途

  • 存储构建信息（如构建时间、版本号）
  • 存储指向日志、监控、仪表板的链接
  • 存储用户的联系信息
  • 存储调试信息
  • 存储准入控制器的配置

控制器

控制器（Controllers）是 Kubernetes 中的控制循环，用于确保集群状态与你声明的期望状态相匹配。控制器通过 API Server 监控集群状态，并根据需要进行调整。

控制器的工作原理

1. 声明期望状态（创建对象）
↓
2. 控制器监控实际状态
↓
3. 对比期望状态和实际状态
↓
4. 如果不一致，执行调整操作
↓
5. 持续监控和循环

常见的控制器类型

控制器 管理的对象 功能
ReplicaSet Pod 确保指定数量的 Pod 副本在运行
Deployment ReplicaSet 管理应用的部署、更新和回滚
StatefulSet Pod 管理有状态应用，维护 Pod 的稳定身份
DaemonSet Pod 确保每个节点上运行一个 Pod 副本
Job Pod 管理一次性任务
CronJob Job 定期执行任务
Node Controller Node 监控节点状态，处理节点故障
重要概念：控制器通过调谐循环（Reconciliation Loop）不断工作。即使你只声明一次期望状态，控制器也会持续监控实际状态，并在需要时进行调整。

Kubernetes API

Kubernetes API 是控制平面的核心组件，是所有组件（内部和外部）与 Kubernetes 集群交互的唯一入口点。它是一个 RESTful API，支持 JSON 和 Protobuf 格式。

API 版本

Kubernetes API 使用版本控制来支持演化和弃用。API 版本分为以下几类：

API 版本组 示例 说明
alpha v1alpha1 早期版本，包含错误，可能不兼容，不建议在生产环境使用
beta v1beta1 已测试，但细节可能变化，适用于非关键生产环境
stable v1 稳定版本，保证向后兼容，推荐在生产环境使用

API 访问方式

  • kubectl：官方的命令行工具，最常用的交互方式
  • 客户端库：官方提供了 Go、Python、Java 等语言的客户端库
  • 直接 REST 调用：可以使用任何支持 HTTP 的工具或语言
  • 自定义控制器：使用 Operator 模式开发自定义控制器
API 优先级和公平性：Kubernetes 1.18 引入了 API 优先级和公平性（APF），用于控制不同类型的 API 请求的优先级，防止某些请求耗尽 API Server 的资源。

kubectl 命令行工具

kubectl 是 Kubernetes 的官方命令行工具，用于与 Kubernetes 集群进行交互。通过 kubectl，你可以部署应用程序、检查和管理集群资源、查看日志等。

基本命令语法

kubectl [command] [TYPE] [NAME] [flags]

常用命令汇总

类别 命令 说明
资源查看 kubectl get nodes 列出所有节点

本章总结

关键要点：
  • Kubernetes 是一个容器编排平台，用于自动化部署、扩展和管理容器化应用
  • Kubernetes 由控制平面和工作节点组成，通过 API Server 进行统一管理
  • Kubernetes 使用对象来表示集群状态，通过控制器确保期望状态与实际状态一致
  • 标签和选择器是组织和选择对象的核心机制
  • 命名空间提供资源隔离和多租户支持
  • kubectl 是与 Kubernetes 集群交互的主要工具
